# BitaP Web担当者マニュアル

このサイトは **「Claudeに文章でお願いする → 自動でサイトが更新される」** 仕組みで運用しています。
プログラミングの知識は不要です。やることは大きく3つだけ。

| やりたいこと | このマニュアルの場所 |
|------------|------------------|
| **お知らせ（news）を更新したい** ←いちばん使う | [1. お知らせの更新](#1-お知らせの更新いちばん使う) |
| **ホーム（トップページ）を更新したい** | [2. ホームの更新](#2-ホームの更新) |
| **デザイン（見た目）を変えたい** | [3. デザインを変えたいとき](#3-デザインを変えたいとき) |

> 公開・反映の仕組み（GitHub / Netlify）について詳しく知りたいときは `web-deploy-manual.md` を見てください。
> このマニュアルは「日々の更新のやり方」に絞って書いています。

---

## はじめに：基本の流れ

どの作業も、考え方は同じ3ステップです。

```
 ① 自分で文章や写真を用意する（content.md を編集 / 写真を images に入れる）
        ↓
 ② Claudeに「反映して、pushして」とお願いする
        ↓
 ③ 数分後、自動でサイトに反映される
```

- **`content.md`** … 人が読み書きする「原稿」ファイル。ここを直すのが担当者の仕事。
- **`index.html`** … Claudeが原稿から自動で作る「表示用」ファイル。**自分では触らない。**
- **push（プッシュ）** … 変更をネット上に送ること。Claudeに頼めばやってくれます。

> 迷ったら、まず Claude に日本語で相談してOKです。「○○したいんだけど、どうすればいい？」でも答えてくれます。

## 1. お知らせの更新（いちばん使う）

お知らせ記事は `news/日付/` というフォルダ単位で管理しています。

```
news/
└── 20260601/              ← 日付フォルダ（YYYYMMDD）
    ├── content.md         ← あなたが書く原稿
    ├── index.html         ← Claudeが自動生成（触らない）
    └── images/            ← この記事で使う写真を入れる場所
```

### STEP 1：記事の「箱」を作ってもらう

日付とタイトルを伝えるだけでOK。Claudeにこう頼みます。

```
お知らせの箱を作って。
日付：2026/06/01
タイトル：【事業事例】〇〇を実施しました
```

→ Claudeが `news/20260601/` フォルダ・空の `content.md`・`images/` を用意します。

### STEP 2：本文を書く（ここが担当者のメイン作業）

`news/20260601/content.md` をテキストエディタで開いて書きます。
**先頭の `---` で囲まれた部分（日付・タイトル・サムネイル）は必ず残してください。**

```
---
date: 2026/06/01
title: 【事業事例】〇〇を実施しました
thumbnail: 20260601_1.jpg
---

ここに本文を書きます。普通に文章を書くだけでOKです。

空行を1つ入れると、段落が変わります。

![](images/20260601_1.jpg)

写真の下に続きの文章を書けます。
```

書き方の見本は **`_template/news-template.md`** に全部載っています（画像1枚・2枚並べ・引用など）。困ったらこれをコピーして使ってください。

#### ◆ 文章を自分で書くのが大変なとき → Claudeに書かせる

きれいな文章にする必要はありません。**箇条書きのメモを渡せばClaudeが記事にしてくれます。**
例えばこんな風に頼みます。

```
20260601の記事の本文を書いて。内容はこのメモを元にして、です・ます調で、
これまでの記事と同じトーンでお願い。最後にお問い合わせフォームの案内も入れて。

メモ：
・2026年3月6日に開催
・主催はアカデミスト株式会社さん
・若手研究者向けの懇親会でケータリングを提供した
・テーマは地中海料理（腸活によい）
・参加者は50名くらい
・好評だった
```

→ Claudeが過去記事の文体に合わせて `content.md` に本文を書き込みます。
**出来上がりを読んで、直したいところがあれば「ここをこう変えて」と追加で頼めばOK。**

> コツ：日時・場所・主催者・人数・テーマ・反応 などの「事実」だけメモすれば十分です。
> 文章の整え方はClaudeに任せられます。

#### ◆ 写真の入れ方

1. 使いたい写真を **`news/20260601/images/` フォルダに入れる**
2. ファイル名は `20260601_1.jpg`、`20260601_2.jpg` … のように分かりやすく（番号順）
3. 本文の中の、写真を出したい位置に1行書く：

```
![](images/20260601_1.jpg)
```

- **サムネイル**（お知らせ一覧に表示される代表写真）は、先頭の `thumbnail:` にファイル名を書きます。
- 写真を**2枚・3枚 横並び**にしたいときは `_template/news-template.md` の「2col」「3col」の書き方を真似してください。
- 写真がない記事は、`thumbnail:` を空欄のままにすればOKです。

### STEP 3：サイトに反映してもらう

原稿と写真が用意できたら、Claudeに頼むだけ。

```
20260601のcontent.mdを見て、index.htmlを更新して、お知らせ一覧にも追加して、pushして
```

→ Claudeが `index.html` を生成し、一覧（`news.json`）にも追加し、pushします。数分でサイトに反映されます。

### お知らせのよくある操作

| やりたいこと | お願いの仕方 |
|-------------|------------|
| 新しい記事を追加 | STEP 1 → 2 → 3 |
| 本文を直す | `content.md` を編集 → STEP 3 |
| 写真を追加 | `images/` に入れる → `content.md` に `![](images/…)` を追記 → STEP 3 |
| 記事を消す | 「20260601の記事を削除して」と頼む |
| 公開順を直したい | 「お知らせの並び順を日付の新しい順にして」と頼む |

## 2. ホームの更新

トップページの原稿は **`content.md`**（`1_web` 直下にあるファイル）です。
ここを直して、Claudeに反映を頼みます。

### 文章を変える

`content.md` を開くと、`## ヒーロー` `## 事業実績` `## 会社概要` `## 代表` … と
セクションごとに分かれています。変えたい箇所の文章を直接書き換えてください。

書き換えたら、Claudeに：

```
ホームのcontent.mdを見て、index.htmlを更新してpushして
```

口頭で頼むこともできます：

```
ホームの会社概要の資本金を「300万円」に変えてpushして
```

### ホームに写真を入れる・差し替える

ホーム用の写真は **`images/` フォルダ**（`1_web` 直下）に入れます。

現在、次の場所が「写真待ち（プレースホルダー）」になっています。

| 場所 | 入れるファイルの例 | 今の状態 |
|------|----------------|---------|
| 代表者の写真 | `images/profile.jpg` | 「差し替え予定」の枠だけ表示中 |
| テレビ出演の画像 | `images/tv_appearance.jpg` | 「差し替え予定」の枠だけ表示中 |

写真を入れる手順：

1. 写真を `images/` フォルダに入れる（例：`profile.jpg`）
2. Claudeに頼む：

```
images/profile.jpg を代表者写真の場所に入れて、index.htmlを更新してpushして
```

> 新しく写真を増やしたいときも同じです。「この写真をホームの○○の下に入れて」と、
> 場所を言葉で伝えればClaudeが配置してくれます。

## 3. デザインを変えたいとき

サイト全体の見た目（色・文字の大きさ・余白・並び方など）は **`style.css`** というファイルで決まっています。
ここも自分で触る必要はなく、**Claudeに「こうしたい」と伝えるだけ**で変えられます。

### いちばん伝わりやすい：参考サイトのURLを渡す

「こういう雰囲気にしたい」という見本のサイトがあれば、そのURLを渡すのが一番確実です。

```
このサイトみたいな雰囲気にしたい：https://example.com
特に、トップの大きな見出しの感じと、全体の余白の広さを参考にして。
うちのサイト（index.html / style.css）をその方向に寄せてpushして
```

> ポイント：URLだけでなく「**どこが好きなのか**」を一言添えると、ねらい通りになりやすいです。
> （例：「色づかい」「文字のフォント」「写真の見せ方」「余白の広さ」など）

### 言葉で雰囲気を伝えてもOK

参考サイトがなくても、イメージを言葉で伝えれば大丈夫です。

```
サイト全体を、もう少し明るくやわらかい印象にしたい。
丸みのあるボタンにして、文字も少し大きめで読みやすくして。pushして
```

### ピンポイントで直す

```
見出しの色を、今のピンクからもう少し落ち着いた緑系に変えて
```
```
お知らせのカードの角を、もっと丸くして
```
```
スマホで見たときに文字が小さいので、少し大きくして
```

### デザインを変えるときのコツ

- **一度に1〜2か所ずつ**お願いすると、結果を確認しながら進められて失敗が少ないです。
- 変えてみて気に入らなければ「**さっきの変更を元に戻して**」と言えば戻せます。
- 大きく変えたいときは、まず「**お試しで変えてみて、見せて**」と頼み、よければ「pushして」と言うと安全です。

## 困ったときは

| 症状 | 対処 |
|------|------|
| 直したのにサイトが変わらない | 反映に数分かかります。少し待ってブラウザを更新（スーパーリロード）。それでもダメなら「ちゃんとpushされてる？」とClaudeに確認 |
| 写真が表示されない | ファイルが正しい `images/` フォルダに入っているか、`content.md` のファイル名と一致しているか確認 |
| やり方が分からない | Claudeに日本語で「○○したいんだけど、どうすればいい？」と聞けばOK |
| 仕組み自体を知りたい | `web-deploy-manual.md`（公開の仕組み全体）を読む |

---

### まとめ：覚えることはこれだけ

1. **原稿（`content.md`）を直す**か、**写真を `images/` に入れる**
2. **Claudeに「反映してpushして」と頼む**
3. **数分待つと、サイトが自動で更新される**

きれいな文章や、難しい操作はClaudeに任せてOK。担当者は「何を伝えたいか」を用意するだけです。
