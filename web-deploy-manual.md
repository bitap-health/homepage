# プログラミングできなくてもOK：Webサイト自動公開マニュアル

Claude Code でサイトを作り、GitHub に置き、Netlify で自動公開する。
**「mainブランチに上げたものが、そのまま本番サイトに反映される」** 仕組みを作るための手順書です。

このマニュアルは、コードを書いたことがない人でも、上から順にやれば一人で完成できるように書いています。

-----

## この仕組みでできること

- Claude にお願いするだけでWebサイト（ランディングページなど）が作れる
- 修正したいときは Claude に直してもらって、GitHub に「push（プッシュ）」するだけ
- push すると Netlify が自動で気づいて、数分後にサイトが更新される
- 独自ドメイン（例：`example.com`）でも公開できる
- https（鍵マーク）も自動でつく

### 全体の流れ（イメージ図）

```
 ① Claude Code でサイトを作る（HTML/CSS/JS）
        ↓
 ② GitHub に push（コードを保管する場所に送る）
        ↓
 ③ Netlify が自動で検知 → 数分でビルド & 公開
        ↓
 ④ サイトに反映完了（独自ドメイン + https）
```

一度この線路を作ってしまえば、次からは「Claude で直す → push する」だけで、勝手にサイトが更新されます。

## 登場するサービスと役割

|役割        |サービス         |費用             |ざっくり何者か           |
|----------|-------------|---------------|------------------|
|サイトを作る    |Claude Code  |Claude利用料のみ    |文章で頼むとコードを書いてくれる相棒|
|コードを保管する  |GitHub       |無料             |コードの保管庫＆履歴管理      |
|公開・自動更新する |Netlify      |無料             |サイトを世界に見せてくれるサーバー |
|住所（独自ドメイン）|XServerドメイン 等|年1,500〜1,600円程度|`〇〇.com` のような名前を取得|

> **メモ：** ここで言う「XServer」は、レンタルサーバー本体ではなく **ドメイン（住所）の取得だけ** に使います。サイトの中身は Netlify が持ちます。ドメインはお名前.com や Cloudflare などでも構いません。

## 事前に用意するもの（アカウント作成）

下の3つを先に作っておきます。どれもメールアドレスがあれば無料で作れます。

1. **GitHub アカウント** … <https://github.com>
1. **Netlify アカウント** … <https://www.netlify.com> （※ GitHubアカウントでそのままログインするのが楽）
1. **Claude** … サイト作成に使用

> ヒント：Netlify は「Sign up with GitHub」で登録すると、後の連携が一段ラクになります。

-----

## STEP 1：Claude Code でサイトを作る

Claude に、作りたいサイトのイメージを文章で伝えます。例えば：

> 「スポーツ系の、力強い雰囲気のランディングページを1枚作って。スクロールすると画像が切り替わる感じで。」

ポイントは次の2つです。

- 完成したファイルの中で、**トップページになるファイルの名前は必ず `index.html` にする**
  （Netlify は `index.html` を「サイトの入口」として自動で読みます。別名だと画面が真っ白になります）
- 凝ったものは後からいくらでも直せるので、まずは「とりあえず1枚」完成させる

> つまずきポイント：過去に、ファイル名が `index.html` でなかったために公開しても何も表示されない、ということがありました。表示されないときはまずファイル名を疑ってください。

-----

## STEP 2：GitHub にコードを置く

「GitHub」はコードの保管庫です。ここに置いたものが、後で Netlify に自動連携されます。

### 2-1. GitHub で新しいリポジトリ（保管箱）を作る

1. <https://github.com> にログイン
1. 右上の「＋」→「New repository」
1. リポジトリ名を入力（例：`lifiya-lp`）
1. **「Add a README file」などのチェックは付けない**（空のまま作るとトラブルが少ない）
1. 「Create repository」をクリック

作成後の画面に、`https://github.com/ユーザー名/lifiya-lp.git` のようなURLが出ます。これを使います。

### 2-2. 作ったサイトを GitHub に送る（初回だけ）

Claude Code に、このようにお願いするのが一番ラクです：

> 「このフォルダを git で初期化して、GitHubの `https://github.com/ユーザー名/lifiya-lp.git` に push して」

Claude が裏で次のようなコマンドを実行してくれます（自分で打ってもOK）：

```bash
git init
git add .
git commit -m "Initial site"
git branch -M main
git remote add origin https://github.com/ユーザー名/lifiya-lp.git
git push -u origin main
```

これで GitHub にサイトのコードが入りました。GitHub のページを開いて `index.html` が見えていれば成功です。

> つまずきポイント：`! [rejected] ... non-fast-forward` というエラーが出たら、GitHub側に先に何か入っている合図です。`git pull origin main` を先に実行してから、もう一度 push してください。

-----

## STEP 3：Netlify につないで自動公開する

ここが「自動で公開される」仕掛けの中心です。

1. <https://www.netlify.com> にログイン
1. 「**Add new site（または Add new project）**」をクリック
1. 「**Import an existing project**」を選ぶ
1. Git プロバイダで「**GitHub**」を選び、アクセスを許可
1. さっき作ったリポジトリ（例：`lifiya-lp`）を選ぶ
1. 公開設定を確認する
- **Branch to deploy（公開するブランチ）**：`main`
- **Build command（ビルドコマンド）**：空欄でOK（HTML/CSSだけの静的サイトなら不要）
- **Publish directory（公開するフォルダ）**：`index.html` がフォルダの一番上にあるなら `.`（ドット）、または空欄
1. 「**Deploy site**」をクリック

数分待つと、`random-name-12345.netlify.app` のようなURLが発行されます。これを開いてサイトが表示されれば **公開成功** です。世界中からアクセスできる状態です。

> この時点で「mainに push → Netlifyが自動でデプロイ」の線路が完成しています。
> Netlify は GitHub に自動で“見張り”を仕掛けるので、以降は push するだけで更新されます。

-----

## STEP 4：独自ドメイン（〇〇.com）をつける ※任意

`random-name.netlify.app` でも公開はできていますが、`example.com` のような独自の住所にしたい場合の手順です。

### 4-1. ドメインを取得する

1. XServerドメイン（<https://www.xdomain.ne.jp）などにアクセス>
1. 欲しい名前（例：`lifiya`）で空きを検索
1. `.com` などを選んで購入

### 4-2. Netlify にドメインを登録する

1. Netlify の対象サイトを開く
1. 左メニュー「**Domain management**」→「**Add a domain**」
1. 取得したドメイン名（例：`lifiya.com`）を入力

### 4-3. ドメイン側でDNSを設定してつなぐ

Netlify がつなぎ方を案内してくれます。方式は主に2つあり、画面の指示に従えばOKです。

- **方式A（ネームサーバー方式）**：Netlify が表示する4つのネームサーバー（`dns1.p0X.nsone.net` のような値）を、ドメイン管理画面の「ネームサーバー設定」に貼り付ける
- **方式B（外部DNS方式 / Aレコード）**：ドメイン管理画面の「DNSレコード設定」で、Netlify が指定するIPアドレス（Aレコード）や CNAME を登録する

> 補足：過去のケースでは、Netlify 画面が外部DNS前提の表示になっていたため **方式B（Aレコード）** で接続しました。Netlify の画面に出ている方式に合わせるのが確実です。

設定後、**数時間〜最大48時間**で反映されます。反映が終わると Netlify 画面の警告マークが消え、https（鍵マーク）も自動でつきます。

## 完成後の「更新のやり方」（毎回これだけ）

公開後にサイトを直したくなったら、次の3ステップだけです。

```
 ① Claude Code でサイトを修正してもらう
        ↓
 ② GitHub に push する
        ↓
 ③ Netlify が自動検知 → 数分でサイトに反映
```

push は、Claude Code に「変更を GitHub に push して」と頼めばやってくれます。
自分で打つ場合は次の3行です：

```bash
git add .
git commit -m "どこを直したかの説明"
git push
```

push したら、Netlify の「Deploys」タブを見れば進行状況（ビルド中→Published）が確認できます。

## 困ったとき（よくあるトラブル）

|症状                              |原因と対処                                                                       |
|--------------------------------|----------------------------------------------------------------------------|
|サイトが真っ白／表示されない                  |トップページのファイル名が `index.html` か確認。違えば `index.html` にリネームして再 push               |
|push が `non-fast-forward` で拒否される|`git pull origin main` を先に実行してから、もう一度 push                                  |
|Netlify がリポジトリを認識しない            |Netlify の Site settings → Build & deploy で Git連携を一度切って繋ぎ直す。GitHub側のアクセス許可も確認|
|独自ドメインがつながらない                   |DNSは反映に最大48時間かかる。設定値（ネームサーバー or Aレコード）が Netlify の案内通りか再確認                   |
|更新したのにサイトが変わらない                 |Netlify の Deploys タブで最新のデプロイが「Published」になっているか確認。ブラウザのキャッシュ削除（スーパーリロード）も試す |

## 用語ミニ辞典（初心者向け）

- **リポジトリ**：コードの保管箱。1サイト＝1リポジトリと考えればOK
- **push（プッシュ）**：手元のコードを GitHub に送ること
- **main（メイン）ブランチ**：本番用の標準の置き場所。ここに入ったものが公開される
- **デプロイ**：作ったものを公開状態にすること
- **ビルド**：コードを公開できる形に組み立てる作業（HTMLだけの静的サイトなら基本不要）
- **DNS / ネームサーバー / Aレコード**：ドメイン（住所）とサーバー（実際の建物）を結びつける設定
