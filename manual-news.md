# お知らせ追加マニュアル

Claudeにお願いするだけで、お知らせを追加できます。このマニュアルを読めば、何を用意してClaudeに何を伝えればいいかがわかります。

---

## ファイル構成のおさらい

```
web/
├── news.json          ← 一覧ページに表示されるカードの情報
└── news/
    ├── 20260420/      ← 日付ごとの記事フォルダ
    │   ├── index.html ← 記事の中身
    │   └── images/    ← この記事で使う画像
    ├── 20260327/
    │   ├── index.html
    │   └── images/
    └── ...
```

---

## お知らせを追加する手順

### ステップ1：画像を用意する（画像がある場合）

使いたい画像ファイルを用意します。
ファイル名は日付を含むものにすると管理しやすいです。

例：`20260601_1.jpg`、`20260601_2.jpg`

### ステップ2：Claudeに伝える

以下の情報をClaudeに伝えてください。

```
お知らせを追加してください。

日付：2026/06/01
タイトル：【事業事例】〇〇を実施しました
サムネイル画像：20260601_1.jpg（一覧カードに表示する画像）
本文：
  〇〇の背景...
  （画像：20260601_2.jpg）
  〇〇の結果...
```

### ステップ3：画像ファイルを所定の場所に保存する

Claudeが「`news/20260601/images/` に画像を入れてください」と指示します。
その場所にファイルを保存してください。

### ステップ4：Claudeにpushしてもらう

「pushして」と伝えると、GitHubに送られ、数分でサイトに反映されます。

## Claudeへの伝え方テンプレート

### パターンA：シンプルなテキストのみ

```
お知らせを追加してください。

日付：2026/06/01
タイトル：【事業事例】〇〇を実施しました
本文：
  〇〇な背景から、〇〇様よりご依頼をいただきました。
  当日は〇〇名が参加し、〇〇を提供しました。
  参加者からは「〇〇」というお声をいただきました。
```

### パターンB：画像と文章を交互に入れる

```
お知らせを追加してください。

日付：2026/06/01
タイトル：【事業事例】〇〇を実施しました
サムネイル画像：20260601_1.jpg

本文の構成：
  ① テキスト「〇〇な背景から...」
  ② 画像：20260601_2.jpg
  ③ テキスト「当日は...」
  ④ 画像：20260601_3.jpg
  ⑤ テキスト「参加者からは...」
```

### パターンC：画像なし・一言お知らせ

```
お知らせを追加してください。

日付：2026/06/01
タイトル：〇〇メディアに掲載されました
本文：〇〇メディアに代表インタビューが掲載されました。
```

---

## 画像についての注意

- 画像は必須ではありません（ない場合はグレーのプレースホルダーが表示されます）
- **サムネイル画像**：一覧ページのカードに表示される画像（1枚）
- **本文画像**：記事の中に入れる画像（何枚でもOK）
- 画像ファイルはClaudeが指示した場所（`news/日付/images/`）に保存してください

---

## よくある質問

**Q. 過去の記事を修正したい**
「`news/20260420/`の記事の本文を修正してください」とClaudeに伝えてください。

**Q. 記事を削除したい**
「`20260420`の記事を削除してください」とClaudeに伝えてください。`news.json`の該当行とフォルダを削除します。

**Q. サムネイル画像だけ後から追加したい**
「`20260430`の記事にサムネイル画像を追加してください。ファイル名は`20260430_1.jpg`です」と伝えてください。

**Q. 記事の順番を変えたい**
`news.json`の並び順がそのまま一覧の表示順になります。Claudeに「〇〇の記事を一番上に移動して」と伝えてください。
