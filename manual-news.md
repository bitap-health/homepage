# お知らせ追加・編集マニュアル

## ファイル構成

```
news/
└── 20260601/
    ├── content.md   ← 人が読み書きするファイル（内容確認・指示出しに使う）
    ├── index.html   ← Claudeが生成するファイル（直接触らない）
    └── images/      ← この記事で使う画像
```

**content.mdのフォーマット：**

```
---
date: 2026/06/01
title: 【事業事例】〇〇を実施しました
thumbnail: 20260601_1.jpg
---

本文テキスト...

![](images/20260601_2.jpg)

続きのテキスト...
```

---

## お知らせを新規追加する

Claudeに以下を伝えるだけでOKです。

```
お知らせを追加してください。

日付：2026/06/01
タイトル：【事業事例】〇〇を実施しました
サムネイル画像：20260601_1.jpg

本文：
  〇〇の背景...
  （画像：20260601_2.jpg）
  〇〇の結果...
```

Claudeが行うこと：
- `news/20260601/` フォルダを作成
- `content.md` を作成
- `index.html` を生成
- `news.json` に追記
- GitHubにpush → Netlify自動反映

画像ファイルは `news/20260601/images/` に保存してください。

---

## 既存記事を編集する

### 方法A：Claudeに口頭で伝える
```
20260601の記事の本文を修正してください。
「〇〇の背景...」の部分を「〇〇〇〇...」に変えてください。
```
Claudeが `content.md` と `index.html` を両方更新します。

### 方法B：content.mdを直接編集してClaudeに反映を頼む
1. `news/20260601/content.md` をテキストエディタで編集
2. Claudeに「20260601のcontent.mdを見てindex.htmlを更新してpushして」と伝える

---

## 現状確認のしかた

記事の内容を確認したいときは `content.md` を開いてください。HTMLより読みやすく、現在の内容がそのまま書かれています。

## よくある操作

| やりたいこと | Claudeへの伝え方 |
|-------------|----------------|
| 記事を追加 | 「お知らせを追加して。日付：〇〇、タイトル：〇〇...」 |
| 本文を修正 | 「20260601の記事の〇〇の部分を〇〇に直して」 |
| 画像を追加 | 「20260601の記事に画像20260601_3.jpgを本文の最後に追加して」 |
| 記事を削除 | 「20260601の記事を削除して」 |
| サムネイルを変更 | 「20260601のサムネイルを20260601_2.jpgに変えて」 |
| pushする | 「pushして」 |
